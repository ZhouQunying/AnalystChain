## ✅ 推荐方案：Markdown Checklist

### 为什么选择Markdown？

1. **版本控制友好** - Git可以追踪变更历史
2. **无需额外工具** - 任何文本编辑器都能用
3. **Cursor原生支持** - 在Cursor中直接编辑
4. **格式简单** - 易于阅读和维护
5. **AI友好** - AI助手可以直接更新

### 与其他方案对比

| 方案 | 优点 | 缺点 | 推荐度 |
|------|------|------|--------|
| **Markdown Checklist** | 简单、版本控制、AI友好 | 功能基础 | ⭐⭐⭐⭐⭐ |
| GitHub Issues | 强大、协作友好 | 需要联网、复杂 | ⭐⭐⭐ |
| Notion/Trello | 可视化好 | 需要外部工具 | ⭐⭐ |
| Taskfile.yml | 自动化强 | 主要用于构建任务 | ⭐⭐ |
| TODO comments | 与代码集成 | 难以全局查看 | ⭐⭐ |

## 🎯 使用方法

### 1. 查看当前阶段

打开 `主任务.md`，找到当前进行中的阶段（标记为 🟡）

### 2. 进入子任务

点击链接进入该阶段的详细子任务文件

### 3. 完成任务时

**方式A：手动勾选（推荐）**
```markdown
- [ ] 未完成的任务
- [x] 已完成的任务  # ← 改成 [x]
```

**方式B：让AI助手更新**
```
"帮我把阶段1的任务3标记为完成"
```

### 4. 更新进度条

```markdown
总进度: ████░░░░░░░░░░░░░░░░ 2/10 (20%)
        ^^^^ 这里的█数量代表完成百分比
```

**计算方法：**
- 每10%一个█
- 例如：20% = ████ (4个█)

## 📊 状态符号说明

### 主任务状态
| 符号 | 状态 | 使用场景 |
|------|------|----------|
| ⏳ | 进行中 | 当前正在执行的阶段 |
| ✅ | 已完成 | 阶段已完成并验证 |
| ⏸️ | 未开始 | 等待前置条件 |
| ⚪ | 未开始 | 尚未开始 |
| 🔴 | 阻塞 | 遇到问题需要解决 |

### 子任务状态
| 符号 | 状态 | Markdown |
|------|------|----------|
| ✅ | 已完成 | `- [x]` |
| ⏳ | 进行中 | `- [ ]` + 状态说明 |
| ⏸️ | 未开始 | `- [ ]` |
| 🔴 | 阻塞 | `- [ ]` + 问题说明 |

### 优先级
| 符号 | 优先级 | 说明 |
|------|--------|------|
| ⭐⭐⭐ | 高 | 必须完成 |
| ⭐⭐ | 中 | 重要但不紧急 |
| ⭐ | 低 | 可选优化 |

## 🔄 日常工作流

### 每天开始工作前

1. **打开主任务** → 查看当前阶段
2. **打开子任务** → 找到今天要做的任务
3. **检查问题追踪** → 确认没有阻塞问题

### 完成一个任务后

1. **更新checkbox** → `[ ]` 改成 `[x]`
2. **更新进度条** → 增加█数量
3. **更新时间记录** → 填写实际用时
4. **记录问题**（如有）→ 添加到问题追踪

### 完成一个阶段后

1. **更新主任务** → 标记阶段为 ✅
2. **更新里程碑** → 记录完成日期
3. **总结经验** → 写入工作日志
4. **开始下一阶段** → 更新状态为 ⏳

## 📝 最佳实践

### ✅ 好的习惯

1. **及时更新** - 完成任务后立即勾选
2. **记录时间** - 填写实际用时，便于估算
3. **记录问题** - 遇到问题马上记录，避免忘记
4. **写工作日志** - 每天结束时总结
5. **定期review** - 每周回顾进度和问题

### ❌ 避免的陷阱

1. **忘记更新** - 做了很多但没记录
2. **过度细化** - 任务拆得太细，维护成本高
3. **状态不准** - 很久没更新，状态失真
4. **只记成功** - 失败的尝试也要记录

## 📊 进度追踪脚本（可选）

如果你想自动化统计进度，我可以帮你写一个Python脚本：

```python
# scripts/check_progress.py
import re
from pathlib import Path

def count_tasks(file_path):
    """统计任务完成度"""
    content = Path(file_path).read_text()
    total = len(re.findall(r'- \[.\]', content))
    done = len(re.findall(r'- \[x\]', content))
    return done, total

# 使用
done, total = count_tasks('docs/tasks/阶段1_知识基础.md')
print(f"阶段1进度: {done}/{total} ({done/total*100:.0f}%)")
```
